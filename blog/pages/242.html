Невозможно расширить сегмент lob

Ошибка:  невозможно расширить сегмент lob в разделе PARUS_MAIN.

Табличное пространство PARUS_MAIN — smallefile  ограничен размеров в 32G.

Решение: добавить в табличное пространство PARUS_MAIN файл данных:

-- добавить в табличное пространство файл
ALTER TABLESPACE PARUS_MAIN ADD DATAFILE 'D:\FRS\PARUS_MAIN2.DAT' SIZE 128M;

Табличное пространство smallfile – это новое название старых табличных пространств. В таких табличных пространствах, может быть несколько файлов данных. Каждый файл данных может быть из 2^22 блоков данных. Таким образом, файл данных в табличном пространстве, использующий блоки данных по 8K, ограничен размеров в 32GB. Табличное пространство может содержать до 1022 файлов данных.

Новое в Oracle 10g, это табличные пространства с большими файлами (bigfile). Такие табличные пространства поострены на одном файле данных или временном файле. Может содержать до 2^32 блоков данных. Такое табличное пространство используя блоки данных по 8K, может быть размером до 32TB.
Такие табличные пространства используются для очень больших баз данных. Большие базы данных содержать сотни файлов данных для чтения или записи, и операции которые должны изменять заголовки файлов данных, например установка контрольных точек, занимают довольно продолжительное время. Если вы уменьшите количество файлов, то операции будут исполняться быстрее.

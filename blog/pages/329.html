ADRCI

Репозиторий автоматической диагностики (Automatic Diagnostic Repository — ADR).

ADR Command Interpreter (ADRCI). Инструмент командной строки для управления диагностической информацией, создания и управления отчетами об инцидентах.

Диагностические данные в 11g очищаются автоматически путем фонового процесса MMON.

Есть два временных атрибутов, которые используются для управления сохранением информации в ADR. Оба атрибута соответствует числу часов, после чего процесс MMON фоне чисток истекшим данных ADR.

LONGP_POLICY (долгосрочные) по умолчанию 365 дней и относится к вещам, как происшествия (Incidents) и предупреждения Health Monitor.

SHORTP_POLICY (краткосрочные) по умолчанию 30 дней и относится к вещи, как файлы трассировки (trace) и файлы ядра (core dump files)

adrci
show control
DIA-48448: This command does not support multiple ADR homes
Если у Вас на хосте более одной БД, то нужно указать с какой базой мы работаем.
show homes

set home diag/rdbms/tester/tester

На примере установим 360 часов (15 дней):

set control (SHORTP_POLICY = 360)
set control (LONGP_POLICY = 360)

Мы также можем вручную очистить с помощью команды purge из ADRCI.
В этом примере мы очищаем все файлы трассировки старше 6 дней (в минутах).

purge -age 8640 -type TRACE

Мы видим, что LAST_MANUPRG_TIME колонка теперь заполнена.
show control

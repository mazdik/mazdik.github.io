ORA-39142: несовместимый номер версии

EXPDP – ORA-39142: incompatible version number 3.1

С помощью EXPDP экспортирую базу с 11.2.0.2 и импортирую дамп в базу 10.2.0.4.

При импорте в базу 10.2.0.4 возникает ошибка:

Соединен с: Oracle Database 10g Enterprise Edition Release 10.2.0.4.0 — 64bit Production
With the OLAP, Data Mining and Real Application Testing options
ORA-39000: неверное задание файла дампа
ORA-39142: несовместимый номер версии 3.1 в дамп-файле «C:\dump\expdpdump.dmp»

Решение: добавить номер версии при экспорте

expdp xxxx/xxxxx@xxx FULL=y directory=dmpdir dumpfile=expdpdump.dmp logfile=expdpdump.log version=10.2
